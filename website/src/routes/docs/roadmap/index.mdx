## Design Principles

The ordering of milestones intentionally matches the
[spec-first philosophy](/docs/architecture/):

1. **Spec before code** — every data model and interaction pattern is written
   as documentation before the implementation exists.
2. **Local-first always** — every feature must work without internet access.
3. **Plugin isolation** — core app never imports plugin code directly;
   all communication goes through the Event Bus.
4. **ADHD empathy** — features that add friction are removed, not iterated on.
